# Initialisation du projet avec Vite

Pour initialiser un projet Vue 3 avec Vite, on utilise la commande suivante dans le terminal :

```bash
npm create vite@latest
```

Donner un nom au projet, ici "client", car je souhaite rattacher ce projet à un backend plus tard.

Ensuite, sélection des fonctionnalités :

- Pinia pour la gestion d'état

Une fois terminé, lancer les commandes suivantes pour installer les dépendances et démarrer le serveur de développement :

```bash
cd client
npm install
npm run dev
```

Le projet Vue 3 avec Vite est maintenant prêt à être développé !

**Note pour un habitué de Svelte** : Dans Vue, le point d'entrée est main.js. Il crée l'application et la monte dans la div#app de ton fichier `index.html`.

## Structure de projet Scalable

Pour que le projet reste propre quand il va grandir, nous allons organiser les dossiers de cette manière. Créer ces dossiers dans `src/` :

```plaintext
src/
├── assets/         # Images, logo, CSS global
├── components/     # Petits composants réutilisables (Button, Card, Navbar, Header, Footer)
├── views/          # Composants "Pages" (HomeView, LoginView, TeamView)
├── stores/         # Gestion de l'état (Pinia)
├── router/         # Configuration du routeur Vue
├── services/       # Fichiers pour les appels API (Axios)
└── layouts/        # Structures de page (ex: avec ou sans Navbar)`
```

Cette organisation permet de séparer clairement les différentes parties de l'application, facilitant ainsi la maintenance et l'évolution du code.

## Installer Tailwind CSS

C'est ici que l'on va découvrir la magie du "Utility First". On installe Tailwind et ses outils :

1. Installer Tailwind CSS via npm :

    ```bash
    npm install tailwindcss @tailwindcss/vite
    ```

2. Modifier le fichier `vite.config.js` pour inclure le plugin Tailwind CSS :

    ```javascript
    import { defineConfig } from 'vite'
    import vue from '@vitejs/plugin-vue'
    import tailwindcss from '@tailwindcss/vite' // <-- Import du plugin Tailwind CSS

    export default defineConfig({
      plugins: [
        vue(), 
        tailwindcss(), // <-- Ajout du plugin Tailwind CSS
    ],
    })
    ```

3. On crée (ou vidé) le fichier `src/assets/main.css` et on ajoute simplement ceci :

    ```css
    @import "tailwindcss";
    ```

4. Enfin, on importe ce fichier CSS dans `main.js` :

    ```js
    import './assets/main.css'

    // ... le reste du code
    ```

Et voilà ! **Tailwind CSS** est maintenant configuré dans ton projet Vue 3 avec Vite. On peut commencer à utiliser les classes utilitaires de Tailwind dans les composants pour styliser l'application rapidement et efficacement.

### Le parallèle Svelte vs Vue (Composition API)

En Svelte 5, on ferait :

```svelte
<script>
  let count = $state(0);
  const increment = () => count++;
</script>
<button onclick={increment}>{count}</button>
```

En Vue 3 (composition API), c'est presque la meme chose :

```vue
<script setup>
import { ref } from 'vue';
const count = ref(0); // ref() est l'équivalent de $state()
const increment = () => count.value++; // .value est requis pour modifier une ref
</script>

<template>
  <button @click="increment">{{ count }}</button>
</template>
```

## ESLint

Pour maintenir un code propre et cohérent, il est recommandé d'installer ESLint. Voici comment faire :

1. Installer ESLint et les plugins nécessaires :
  
    ```bash
    npm install --save-dev eslint eslint-plugin-vue
    ```

2. Initialiser ESLint dans le projet :

    ```bash
    npx eslint --init
    ```

3. Configurer ESLint avec le fichier `eslint.config.js` :

    ```js
    import js from "@eslint/js";
    import globals from "globals";
    import pluginVue from "eslint-plugin-vue";
    import { defineConfig } from "eslint/config";

    export default defineConfig([
      {
        files: ["**/*.{js,mjs,cjs,vue}"],
        plugins: { js },
        extends: ["js/recommended"],
        languageOptions: {
          globals: {
            ...globals.browser,
            ...globals.node,
          },
        },
      },
      {
        rules: {
          // Force l'utilisation de guillemets doubles
          quotes: [
            "error",
            "double",
            {
              avoidEscape: true, // Permet d'éviter les erreurs si des guillemets simples sont nécessaires dans la chaîne
              allowTemplateLiterals: true, // Permet l'utilisation de littéraux de gabarits
            },
          ],
        },
      },
      pluginVue.configs["flat/essential"],
      {
        // Configuration spécifique pour les fichiers Vue
        files: ["**/*.vue"],
        rules: {
          // Force les guillemets doubles dans les templates Vue
          "vue/html-quotes": [
            "error",
            "double",
            {
              avoidEscape: true,
              allowTemplateLiterals: true,
            },
          ],
        }
      }
    ]);
    ```
