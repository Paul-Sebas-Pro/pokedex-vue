# Création du service API avec Axios

Nous allons maintenant connecter le frontend à notre API backend en utilisant **Axios**, une bibliothèque populaire pour effectuer des requêtes HTTP.
Nous allons créer un service d'appel API propre et l'utiliser pour gérer la connexion (login)

## 1. Installer Axios

Axios est plus puissant que la fonction native `fetch`car il permet de configurer une "instance" avec une URL de base et d'intercepter les requêtes pour y ajouter automatiquement le token.

```bash
npm install axios
```

## 2. Créer le service API

On crée une instance centralisée d'Axios. C'est ici que la magie opère : si un token existe dans Pinia, on l'ajoute dans les headers de chaque requête automatiquement.

Dans le dossier `src/services`, crée un fichier `api.js` :

```js
import axios from "axios";
import { useAuthStore } from "../stores/auth";

const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL || "http://localhost:3000",
});

// Intercepteur : avant chaque requête, on injecte le token
api.interceptors.request.use((config) => {
  const authStore = useAuthStore();
  if (authStore.token) {
    config.headers.Authorization = `Bearer ${authStore.token}`;
  }
  return config;
});

export default api;
```

## 3. Implémenter le Login

Nous allons créer un formulaire moderne avec Taildwind et utiliser notre service API.

**Note pour Svelte** : Le `bind:value` de Svelte devient `v-model` en Vue.

Dans le fichier `src/views/LoginView.vue` :

```vue
<script setup>
import { ref } from "vue";
import { useAuthStore } from "../stores/auth";
import { useRouter } from "vue-router";
import api from "../services/api";

const authStore = useAuthStore();
const router = useRouter();

const email = ref("");
const password = ref("");
const error = ref(null);
const isLoading = ref(false);

const handleLogin = async () => {
  isLoading.ref = true;
  error.value = null;
  
  try {
    const response = await api.post("/login", {
      email: email.value,
      password: password.value
    });

    // On stocke les infos dans Pinia
    authStore.setToken(response.data.token);
    authStore.setUser(response.data.user);

    // Redirection vers l'accueil
    router.push("/");
  } catch (err) {
    error.value = err.response?.data?.error || "Identifiants incorrects";
  } finally {
    isLoading.value = false;
  }
};
</script>

<template>
  <div class="max-w-md mx-auto mt-20 p-6 bg-white rounded-xl shadow-lg border border-slate-100">
    <h1 class="text-2xl font-bold text-slate-800 mb-6 text-center">Connexion</h1>

    <form @submit.prevent="handleLogin" class="space-y-4">
      <div>
        <label class="block text-sm font-medium text-slate-700">Email</label>
        <input v-model="email" type="email" required
          class="w-full mt-1 p-2 border border-slate-300 rounded-lg focus:ring-2 focus:ring-blue-500 outline-none">
      </div>

      <div>
        <label class="block text-sm font-medium text-slate-700">Mot de passe</label>
        <input v-model="password" type="password" required
          class="w-full mt-1 p-2 border border-slate-300 rounded-lg focus:ring-2 focus:ring-blue-500 outline-none">
      </div>

      <p v-if="error" class="text-red-500 text-sm">{{ error }}</p>

      <button type="submit" :disabled="isLoading"
        class="w-full bg-blue-600 text-white py-2 rounded-lg font-semibold hover:bg-blue-700 transition disabled:bg-blue-300">
        {{ isLoading ? 'Connexion...' : 'Se connecter' }}
      </button>
    </form>
  </div>
</template>
```

## 4. Protéger les routes

On ne veut pas que l'utilisateur accède à certaines pages s'il n'est pas connecté. On utilise un ***"navigation guard"*** de Vue Router pour vérifier cela avant chaque changement de route.

Pour protéger les routes, on procède en deux temps :

1. On marque les routes qui demandent une connexion (via la propriété meta).

2. On crée un garde (beforeEach) qui vérifie ce marquage avant chaque changement de page.

Modifier le fichier `src/router/index.js` :

```js
// ... imports
import { useAuthStore } from '../stores/auth';

/const router = createRouter({
  history: createWebHistory(), // Utilise l'historique du navigateur (URLs propres)
  routes: [
    {
      path: "/",
      name: "home",
      component: HomeView
    },
    {
      path: "/login",
      name: "login",
      component: LoginView
    },
    {
      path: "/teams",
      name: "teams",
      // on utilise le lazy-loading (import dynamique) pour cette page
      component: () => import("../views/TeamsView.vue"),
      // Etape 1 : On marque cette route comme protégée
      meta: { requiresAuth: true }
    }
  ]
});

// Etape 2 : Le Garde de navigation (Navigation Guard)
// Cette fonction s'execute à chaque fois qu'on change d'URL
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore();

  // "to.matched" vérifie si la route (ou ses parents) possède "requiresAuth"
  const isProtected = to.matched.some(record => record.meta.requiresAuth);

  if (isProtected && !authStore.isAuthenticated) {
    // Si la route est protégée et qu'on n'est pas connecté -> redirection login
    next({ name: "login" });
  } else if (to.name === "login" && authStore.isAuthenticated) {
    // Optionnel : si on est déjà connecté, on empêche d'aller sur la page login
    next({ name: "home" });
  } else {
    // Sinon, on laisse passer
    next();
  }
});

export default router;
```

### Explications

- `meta: { requiresAuth: true }` : C'est comme si tu ajoutais une variable personnalisée à ta route. Tu peux y mettre ce que tu veux (un titre de page, des rôles admin, etc.).

- `router.beforeEach` : Imagine que c'est un middleware qui s'exécute juste avant le rendu de la page.

- `next()` : C'est la fonction cruciale. Si tu ne l'appelles pas, la navigation est bloquée.
  - `next()` : Tout va bien.
  - `next("/login")` : Stop ! Va là-bas à la place.

### Pourquoi c'est "Pro" ?

1. Sécurité : L'intercepteur Axios garantit que chaque appel vers ton backend sera authentifié sans que tu aies à y repenser.

2. UX : Le bouton de login change d'état (isLoading) et les erreurs sont affichées proprement.

3. Lazy Loading : Pour /teams, j'ai utilisé une fonction () => import(...). Cela signifie que le code de cette page n'est chargé que si l'utilisateur y va. Gain de performance !
