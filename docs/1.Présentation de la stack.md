# Présentation de la stack

Pour l'application Pokédex, nous allons utiliser une stack moderne, rapide et standard dans l'industrie du développement web. Voici les principaux composants de cette stack :

| Outi         | Rôle                                                                                                                       | Equivalent Svelte (conceptuel)         |
| ------------ | -------------------------------------------------------------------------------------------------------------------------- | -------------------------------------- |
| Vite         | Le moteur de build ultra-rapide qui sert ton code en local et le compile pour la prod.                                     | Vite (également utilisé par SvelteKit) |
| Vue 3        | Le framework. On utilisera la Composition API (`<script setup>`), qui ressemble beaucoup à l'esprit des Runes de Svelte 5. | Svelte 5                               |
| Vue Router   | Gère la navigation entre tes pages (Home, Login, Team) sans recharger le navigateur.                                       | SvelteKit Routing                      |
| Pinia        | Le magasin d'état global (Store). C'est là qu'on stockera le Token JWT et les infos de l'utilisateur.                      | Svelte Stores / Runes exportées        |
| Tailwind CSS | Framework CSS utilitaire. Au lieu d'écrire du CSS dans <style>, on ajoute des classes directement sur le HTML.             | CSS classique ou Tailwind              |
| Axios        | Une bibliothèque pour faire nos appels à ton API REST de façon plus simple que `fetch`.                                    | Fetch API                              |

Cette stack est conçue pour être performante, facile à apprendre et largement adoptée dans l'industrie, ce qui facilitera la maintenance et l'évolution de ton application Pokédex.
